# Hero-видео на главной — как заменить и какие параметры

## Как заменить (без кода)
1. Открой главную с редактором: `https://allclean.pages.dev/?lgedit` → введи пароль.
2. Наведи на видео в шапке → появится бейдж **🎬 видео** → выбери новый файл.
3. Спросит постер (картинку-превью, видна пока видео грузится) — рекомендуется загрузить.
4. Нажми **🚀 Опубликовать** → сайт пересоберётся (~1–2 мин). Обнови с Ctrl+Shift+R.

Файл грузится в Supabase storage и применяется при пересборке (индексируется, не клиентский хак).
Лимит в редакторе: **30 МБ** (для веба нужно гораздо меньше — см. ниже).

## Параметры видео для быстрой загрузки (важно для скорости и SEO/CWV)
Hero-видео — декоративный фон (без звука, автоплей, цикл). Цель — маленький файл и быстрый старт.

| Параметр | Рекомендация |
|---|---|
| **Формат** | MP4 (H.264) — основной; опц. WebM (VP9) для меньшего веса |
| **Разрешение** | 1280×720 (720p) достаточно для фонового видео; максимум 1920×1080 |
| **Длительность** | 8–15 секунд, бесшовный цикл |
| **Битрейт** | 1–2.5 Мбит/с (видео без звука) |
| **Размер файла** | **цель ≤ 3–5 МБ**, жёсткий потолок 30 МБ |
| **FPS** | 24–30 |
| **Звук** | удалить (видео muted, звук не нужен — только вес) |
| **Постер** | JPG/WebP, тот же кадр, ≤ 150 КБ — показывается мгновенно |

### Как сжать (бесплатно)
- Онлайн: **Handbrake** (пресет «Web / Vimeo YouTube 720p30»), CloudConvert, Clideo.
- FFmpeg (если есть): 
  ```
  ffmpeg -i input.mp4 -vf "scale=1280:-2,fps=30" -c:v libx264 -crf 26 -preset slow -an -movflags +faststart hero.mp4
  ```
  `-an` убирает звук, `-movflags +faststart` — видео начинает играть до полной загрузки (важно!).
- Постер из кадра: `ffmpeg -i hero.mp4 -ss 0 -vframes 1 -q:v 4 poster.jpg`

### Почему это влияет на скорость/SEO
- Большое видео грузится долго → плохой LCP/INP → ниже в выдаче. Маленький файл + постер = быстрый первый экран.
- `faststart` и постер дают мгновенную «картинку», пока видео догружается.
- Видео отдаётся с CDN Supabase; постер виден сразу.
